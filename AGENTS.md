# hermes-feishu-zh 开发规则

## 铁律（必须遵守）

1. **先测试后写码** — 写代码前先写测试，没有失败测试就不写实现
2. **完成前必须验证** — 声称完成前运行 verify.ps1，没有通过就不能说完成
3. **中文优先** — 所有用户可见内容用中文，代码/命令/路径保持英文
4. **备份再改** — 修改 config.yaml 或源码前必须备份

## 工作流程

- 收到任务 → 检查是否有匹配 skill → 有就用，没有就直接做
- 简单任务直接执行，不加流程
- 复杂任务：计划 → 实现 → 验证 → 报告

## 禁止事项

- ❌ 不提交 API key 或敏感信息
- ❌ 不跳过验证步骤
- ❌ 不在没有备份的情况下修改生产配置
- ❌ 不写超过 300 行的规则文件
